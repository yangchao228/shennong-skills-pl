# v0.1.0-alpha

Release date: 2026-06-10

This is the first alpha release of Skills Manager, a local-first web console for
managing, testing, versioning, and evolving `SKILL.md` files.

## Positioning

Skills Manager is not a public skill marketplace or a cloud workspace. It is a
local skill asset workbench for people who already maintain their own skills and
want better structure, tests, baselines, diffs, and rollback.

## Highlights

- Local web UI for browsing and inspecting a skills directory
- Structure health checks grouped into blocking, risk, and optimization issues
- External manager metadata directory, so managed skill directories are not
  polluted by test cases, versions, caches, or evolution logs
- Baseline snapshot, baseline diff, version history, and restore flow
- Protected skill roots with explicit writeback confirmation
- Evolution candidate flow: generate candidate, inspect diff, then apply
- In-page confirmation modal for risky writeback actions
- AI provider layer supporting:
  - Ollama
  - Anthropic
  - OpenAI Codex API through the OpenAI Python SDK
  - Local Codex CLI through `codex exec`
- Local smoke and E2E tests in CI

## Quick Start

```bash
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt
./.venv/bin/python app.py
```

Open:

```text
http://localhost:7890
```

To point at a custom skills directory:

```bash
SKILLS_PATH=/path/to/skills ./.venv/bin/python app.py
```

## AI Provider Examples

Use local Ollama:

```bash
export AI_PROVIDER=ollama
export OLLAMA_BASE_URL=http://127.0.0.1:11434
export OLLAMA_MODEL=glm4:latest
```

Use Anthropic:

```bash
export AI_PROVIDER=anthropic
export ANTHROPIC_API_KEY=sk-ant-...
export ANTHROPIC_MODEL=claude-sonnet-4-6
```

Use OpenAI Codex API through the Python SDK:

```bash
export AI_PROVIDER=openai_codex
export OPENAI_API_KEY=sk-...
export OPENAI_CODEX_MODEL=gpt-5.2-codex
```

Use local Codex CLI:

```bash
export AI_PROVIDER=codex
export CODEX_COMMAND=codex
export CODEX_MODEL=gpt-5
```

If your local Codex CLI needs a proxy:

```bash
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
export ALL_PROXY=socks5://127.0.0.1:7891
```

## Security Notes

- The default host is `127.0.0.1`.
- The app currently has no authentication layer.
- Do not expose it directly to the public internet.
- Keep API keys in local environment variables or `.env`.
- Use `SKILLS_MANAGER_PROTECTED_ROOTS` for installed or shared skill roots.
- Writeback actions can overwrite a real `SKILL.md`; protected roots require
  explicit confirmation.

See `SECURITY.md` for details.

## Known Limitations

- Alpha release: expect rough edges in UI and setup.
- No multi-user auth, permission model, or hosted workspace.
- Test case editing is still mostly JSON-oriented.
- Browser-level E2E is manual; CI currently covers Flask route E2E through
  `scripts/e2e_local.py`.
- Real model quality depends on your selected provider and local configuration.

## Suggested Trial Scope

Use this release with a small local skill directory first. Recommended trial:

1. Point `SKILLS_PATH` at a copy of 2-3 real skills.
2. Inspect structure health checks.
3. Generate or write 1-2 tests for one verifiable skill.
4. Run tests and save a baseline.
5. Modify `SKILL.md` locally.
6. Compare with baseline diff.
7. Try candidate evolution on a protected root and inspect the candidate diff
   before applying it.
